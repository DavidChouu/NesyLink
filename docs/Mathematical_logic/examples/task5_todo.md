**总体结构**

将 Task5 改成“三层策略”，而不是让单个 FSM 同时决定全局路线、局部移动和战斗。

```
CNN 像素帧 + safe_info
        ↓
符号观测与事件推断层
        ↓
全局房间图 / 风险记忆 / 宏观规划器
        ↓
当前宏目标：去某房间、开某宝箱、过某出口
        ↓
局部 BFS + 像素对齐 + 战斗/盾牌执行器
```

**第一层：观测与记忆**

继续只从允许的信息建立内部状态：

- 当前房间的相对坐标：仍由“边界持续推进 + 下一帧出生点跳转”推断。
- 每个房间记忆：墙、宝箱、按钮、出口类型、静态阻挡、已开宝箱、已踩按钮、已探索出口。
- 房间危险摘要：看见过的怪物数量、怪物最近位置、陷阱位置、该房间最近一次受伤/阻塞情况。
- 收益摘要：通过 `last_reward` 与物品栏变化推断：
  - 是否拿到钥匙；
  - 是否成功开箱；
  - 该宝箱是否提供治疗；
  - 是否消耗钥匙开门；
  - 是否发生伤害、陷阱或怪物击退。
- 生存预算：不读取隐藏 HP，而是维护估计值：
  - 初始预算设为已知初始生命值；
  - 每 200 个 Agent 动作预扣一次；
  - `last_reward` 出现伤害特征时扣减；
  - 出现 `agent_healed` 对应的高奖励特征时增加。

这个预算不是作弊，因为完全由自己的动作计数和允许的 `last_reward` 推断。

**第二层：全局房间图**

将房间抽象为图：

```
RoomNode:
  已开宝箱、未开宝箱、是否发现治疗收益、按钮状态、风险等级

RoomEdge:
  出口方向、门类型、是否已通过、是否需要钥匙/按钮、
  估计通行代价、最近实际通行耗时、危险代价
```

每次换房后更新边的真实代价，而非一直相信理论 BFS 距离。例如一次出口被怪物干扰、反复撞墙，就提高该边的经验代价。

这样策略学到的是“这个出口从这里过去通常要 150 帧，而且有一只追踪怪”，不是“东边一定有治疗宝箱”。

**第三层：宏观规划器**

宏目标不再是“当前房间最近的物体”，而是候选任务链，例如：

- 开当前可达宝箱；
- 去未知出口探索；
- 去已发现、但尚未处理的房间；
- 去已知治疗宝箱；
- 返回一个有未开宝箱的已访问房间；
- 清理一个确实阻塞宏路线的怪物。

每次重规划时，对候选宏目标计算：

```
总代价 =
  当前房间到出口/目标的 BFS 距离
+ 估计换房成本
+ 目标房间内到宝箱的预估距离
+ 预计怪物风险代价
+ 已观察到的卡墙/失败交互惩罚
+ 周期扣血风险惩罚
```

优先级建议：

1. 预计生命预算不足以完成高风险支路时，优先已发现的治疗机会。
2. 可达未开宝箱优先，但前提是完成后仍能安全返回或继续探索。
3. 锁门只在持钥匙且其后续收益/安全性优于其他分支时优先。
4. 未探索出口作为兜底探索，不再仅由固定方向顺序决定。
5. 当剩余预算不足时，选择“完成所有已知宝箱的最短任务链”，减少探索。

这能解决 `spatial_b` 的关键难题：策略会比较“先去西房会不会在无治疗状态下穿过两只怪”和“先通过锁门获得治疗再去西房”的整个链条，而不是只比较眼前出口距离。

**第四层：局部执行器**

保留现有 BFS，但调整职责：

- BFS 只负责到当前宏目标的最短可行路径。
- `action_blocked` 立即写入局部边阻挡记忆，重算路径。
- 开箱必须由 `last_reward` 或物品栏变化确认，不能仅凭相邻 tile 标记成功。
- 怪物策略分成三种明确模式：
  - `avoid`：有绕行路时避开；
  - `pass`：短路径穿过危险区时一次举盾，继续原路径；
  - `clear`：怪物堵住唯一通道、宝箱交互位或出口时才连续攻击至不可见。
- 不允许“每隔一帧举盾”。盾牌动作必须附带明确的恢复动作和冷却窗口。
- 进入新房间后，先做一次风险评估，再提交长移动队列。

**重规划触发条件**

以下情况必须清空局部队列，回到宏规划器：

- 成功换房；
- 成功开箱、拿钥匙、治疗、开门；
- `last_reward` 表明碰撞阻塞、伤害、陷阱；
- 怪物进入当前路线的危险半径；
- 当前目标在视觉中消失或类别变化；
- 连续若干帧没有空间进展；
- 接近下一次周期扣血且估计生命预算紧张。

**实施顺序**

1. 先将现有 `RoomMemory` 扩展为“房间图节点 + 边经验代价 + 收益/风险记录”。
2. 加入从 `last_reward` 推断伤害、治疗、阻塞和动作耗时的统一事件更新函数。
3. 编写宏目标和候选路线评分函数，但先不动底层 BFS。
4. 把当前固定出口顺序替换成“约束满足后按全局评分选择”。
5. 将盾牌/攻击改为有限状态战斗子策略，消除重复举盾。
6. 每完成一项，回归测试基础局，再逐个测试 `spatial_a`、`spatial_b`、`spatial_c`。
7. 最后用正式 CNN 接口运行多 seed 测试，记录成功率、平均步数、平均 reward 和失败原因。

这条路线比继续微调当前局部启发式更有希望让三个空间局都通过。
